# Sample reports

Reference output from real `./harden.py all` runs, committed to the
repo so readers can see what the framework produces without
provisioning a cluster.

Each subdirectory is one full pipeline run, named for the topology and
date.

## What's included

Per run, the small human-readable artifacts:

- `baseline.md` — pre-hardening kube-bench + kubescape findings
- `post-hardening.md` — post-hardening findings
- `delta.md` — score before/after summary table
- `scores.json` — machine-readable counts and percentages

Raw scanner JSON (`kube-bench.json`, `kubescape.json`) is **not**
committed — both are multi-MB blobs and the same data is summarised
in the Markdown reports above. Run the pipeline yourself to regenerate
them.

## Reading a sample

Start with `delta.md` (one-line scoreboard), then `baseline.md` for
the unhardened starting point, then `post-hardening.md` for what
Tier 1 + Tier 2 fixed and what's left for Tier 3 manual work.

## Reproducing

The DigitalOcean sample was generated by the exact runbook in
[`docs/SETUP-STANDALONE.md`](../../docs/SETUP-STANDALONE.md), using
`scripts/standalone-bootstrap.sh` with `RUN_HARDEN=1`. No manual
tweaks.
